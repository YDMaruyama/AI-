# 03. 日報アシスト

## 1. 概要

### 1.1 この機能が解決する課題

就労継続支援事業所（A型/B型）における日報作成には、以下のような深刻な課題が存在する。

#### 現状の課題一覧

| # | 課題 | 具体的な問題 | 影響 |
|---|------|-------------|------|
| 1 | **紙・Excelでの運用** | 紙の日報用紙に手書き、またはExcelテンプレートに入力。PC操作が苦手なスタッフは30分以上かかることもある | 業務時間の圧迫、紙の紛失リスク、検索不可能 |
| 2 | **書き忘れ・提出漏れ** | 業務終了間際に慌てて書く、翌日以降にまとめて書く。記憶が曖昧になり記録の質が低下 | 正確な記録が残らない、監査対応で困る |
| 3 | **文章力の個人差** | 詳細に書くスタッフと「特に問題なし」だけのスタッフの差が大きい。利用者の様子など重要な情報が欠落 | 引き継ぎ不備、利用者支援の質のばらつき |
| 4 | **社長・管理者の確認負荷** | 全スタッフの日報を1枚1枚確認する必要がある。重要な情報の見落としリスク | 緊急対応の遅れ、経営判断に必要な情報が届かない |
| 5 | **検索・分析の困難さ** | 「先月のAさんの体調に関する記録」を探すのに過去の日報を全ページめくる必要がある | 利用者支援計画の作成が困難、行政報告に時間がかかる |
| 6 | **フォーマットの不統一** | スタッフごとに書き方が異なり、必要項目の漏れが発生 | 情報の網羅性が担保できない |

#### 背景：就労継続支援事業所特有の事情

就労継続支援事業所では、以下の記録が法的に求められる。

- **サービス提供記録**: 利用者ごとの支援内容・様子の記録（指定基準省令）
- **日々の業務記録**: スタッフの業務内容、引き継ぎ事項
- **ヒヤリハット・事故報告**: 利用者の体調変化やトラブルの即時記録

これらの記録は行政の実地指導（監査）で確認されるため、「書いていない」は許されない。しかし現場のスタッフは支援業務で手一杯であり、記録に割ける時間は限られている。

### 1.2 AIが対話形式でアシストする価値

AI秘書がLINE上の会話を通じて日報作成をアシストすることで、以下の価値を提供する。

| 価値 | 具体的な内容 |
|------|-------------|
| **作成時間の大幅短縮** | AIが質問→スタッフが回答→AIが文章整形。5分以内で完了（従来30分） |
| **情報の網羅性を保証** | AIが必ず5つの質問（作業内容・利用者の様子・外部やり取り・引き継ぎ・その他）を順に聞くため、漏れが発生しない |
| **文章品質の均一化** | スタッフの回答が「問題なし」だけでも、AIが適切な文章に整形。逆に冗長な場合は要点を抽出 |
| **リアルタイム共有** | 日報提出と同時に社長に要約通知。緊急キーワード検知時は即時アラート |
| **即座の検索・分析** | 社長が「3月の田中さんの日報まとめて」と聞くだけでAIが集約・要約 |
| **ペーパーレス化** | 紙の日報が不要になり、保管スペース・紛失リスクから解放 |
| **心理的ハードルの低下** | LINEでチャットするだけなので、PCが苦手なスタッフでも負担を感じない |

### 1.3 機能スコープ

```
日報アシスト機能
├── 日報作成アシスト（対話形式）
│   ├── 5問の質問テンプレート
│   ├── 回答の自動整形（Claude API）
│   ├── 確認→修正→保存フロー
│   └── 会話ログの保存
├── 日報閲覧
│   ├── 自分の過去の日報を見る
│   ├── 社長：全スタッフの日報を見る
│   └── 管理者：担当スタッフの日報を見る
├── 日報検索
│   ├── 日付指定検索
│   ├── 人名指定検索
│   ├── キーワード検索
│   └── 期間まとめ（週次/月次サマリー）
├── 通知
│   ├── 提出時の社長通知
│   ├── 未提出リマインド（17:00）
│   ├── 1日のサマリー（18:00）
│   └── 緊急キーワードアラート
└── 管理
    ├── 日報の修正
    ├── 日報のステータス管理
    └── 日報テンプレートの管理
```

## 2. ユーザーストーリー

### 2.1 スタッフ：LINEで会話するだけで日報が完成する

```
スタッフとして、
業務終了後にLINEで「日報書く」と送ると、
AIが順番に質問してくれて、
それに答えるだけで整った日報が自動的に完成し、
社長にも自動で共有されるので、
日報作成の負担がほとんどなくなる。
```

**受入条件:**
- [ ] 「日報」「日報書く」「日報を書く」「にっぽう」のいずれかで日報作成モードに入る
- [ ] AIが5つの質問を順番に提示する
- [ ] 各質問に対してテキストで回答できる
- [ ] 「特になし」「なし」等の回答も受け付ける
- [ ] 全質問完了後、AIが回答を整形して確認メッセージを表示する
- [ ] 確認OKで保存、修正があれば修正フローに入る
- [ ] 保存完了のメッセージが届く
- [ ] 自分の過去の日報を「昨日の日報見せて」等で閲覧できる

### 2.2 管理者：スタッフの日報を確認し、自分も日報を書ける

```
管理者として、
自分の日報をスタッフと同じフローで書けるとともに、
担当するスタッフの日報を確認でき、
未提出のスタッフにリマインドを送れるようにしたい。
```

**受入条件:**
- [ ] 管理者自身も日報作成フローを利用できる（スタッフと同一フロー）
- [ ] 「○○さんの日報」で担当スタッフの日報を閲覧できる
- [ ] 「今日の日報状況」で担当スタッフの提出状況一覧を確認できる
- [ ] 未提出スタッフに対して「○○さんにリマインドして」でリマインドを送れる
- [ ] 管理者が閲覧できるのは担当スタッフの日報のみ（他チームの日報は見えない）

### 2.3 社長：日報の検索・確認・分析が自由自在

```
社長として、
「3月28日の日報」と送るだけで全スタッフの日報を取得でき、
日付・人名・キーワードで自在に検索でき、
週次・月次のまとめも一言で取得したい。
```

**受入条件:**
- [ ] 「3月28日の日報」で当日の全スタッフの日報サマリーを取得
- [ ] 「田中さんの今週の日報」で特定スタッフの期間指定日報を取得
- [ ] 「利用者Aの体調」で関連する日報をキーワード検索
- [ ] 「今月の日報まとめ」で月次サマリーレポートを生成
- [ ] 「今週で引き継ぎ事項があった日報」で条件付き検索
- [ ] 日報提出時にLINEで要約通知を受け取れる
- [ ] 「詳しく」と返すと全文を閲覧できる
- [ ] 未提出者がいる場合、夕方に自動通知を受け取れる

### 2.4 追加ユーザーストーリー

#### 2.4.1 スタッフ：過去の日報を修正したい

```
スタッフとして、
昨日の日報に記載漏れがあった場合、
「昨日の日報を修正したい」と送ると修正フローに入り、
修正箇所だけ書き直せるようにしたい。
```

#### 2.4.2 社長：日報から自動でタスクを抽出したい

```
社長として、
日報の引き継ぎ事項や「要対応」の記述から、
AIが自動でタスクを抽出して提案してくれることで、
対応漏れを防ぎたい。
（※ 04_タスク自動生成 機能として実装）
```

## 5. データモデル

### 5.1 daily_reports テーブル

```sql
-- ============================================================
-- 日報テーブル
-- 就労継続支援事業所のスタッフが毎日提出する日報を格納する
-- ============================================================

CREATE TABLE daily_reports (
    -- === 主キー ===
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- === 外部キー ===
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
    -- 削除時はRESTRICTで保護（日報が存在するユーザーは削除不可）

    -- === 日付 ===
    report_date DATE NOT NULL,
    -- 日報の対象日。業務を行った日を指す（提出日ではない）
    -- 例：3/28の業務について3/29に書いた場合、report_dateは3/28

    -- === 作業内容 ===
    work_content JSONB NOT NULL DEFAULT '[]'::jsonb,
    -- 構造例:
    -- [
    --   {
    --     "time_period": "午前",
    --     "content": "利用者3名とYouTubeサムネイル作成作業を実施",
    --     "participants": ["利用者A", "利用者B", "利用者C"]
    --   },
    --   {
    --     "time_period": "午後",
    --     "content": "新規案件の動画編集指導を実施",
    --     "participants": ["利用者D", "利用者E"]
    --   }
    -- ]

    -- === 利用者の様子 ===
    client_notes JSONB NOT NULL DEFAULT '[]'::jsonb,
    -- 構造例:
    -- [
    --   {
    --     "client_name": "利用者A",
    --     "note": "集中して作業に取り組めていた。良好。",
    --     "alert_level": "normal",
    --     "tags": ["良好", "集中"]
    --   },
    --   {
    --     "client_name": "利用者B",
    --     "note": "体調不良の様子あり。午後から早退。",
    --     "alert_level": "warning",
    --     "tags": ["体調不良", "早退"]
    --   }
    -- ]
    -- alert_level: "normal" | "warning" | "critical"

    -- === 外部とのやり取り ===
    external_contacts TEXT DEFAULT '',
    -- 自由テキスト。外部機関・家族等とのやり取りを記録
    -- 空文字列は「特になし」を意味する

    -- === 引き継ぎ事項 ===
    handover TEXT DEFAULT '',
    -- 翌日以降のスタッフへの引き継ぎ事項
    -- 空文字列は「特になし」を意味する

    -- === その他 ===
    other_notes TEXT DEFAULT '',
    -- 上記カテゴリに該当しないメモ・気づき
    -- 空文字列は「特になし」を意味する

    -- === AI整形後のサマリー ===
    summary TEXT NOT NULL DEFAULT '',
    -- Claude APIで整形された日報の全文テキスト
    -- LINE上で表示する最終版の文面をそのまま格納

    -- === 生の会話ログ ===
    raw_conversation JSONB NOT NULL DEFAULT '[]'::jsonb,
    -- LINE上のAIとの全会話ログを時系列で保存
    -- 構造例:
    -- [
    --   {
    --     "role": "user",
    --     "content": "日報書く",
    --     "timestamp": "2026-03-28T17:00:00+09:00"
    --   },
    --   {
    --     "role": "assistant",
    --     "content": "お疲れさまです！...",
    --     "timestamp": "2026-03-28T17:00:01+09:00"
    --   },
    --   ...
    -- ]

    -- === ステータス ===
    status VARCHAR(20) NOT NULL DEFAULT 'draft'
        CHECK (status IN ('draft', 'submitted', 'revised', 'archived')),
    -- draft: 作成中（一時保存）
    -- submitted: 提出済み
    -- revised: 修正済み（提出後に修正された）
    -- archived: アーカイブ（年度末等の一括処理用）

    -- === メタデータ ===
    version INTEGER NOT NULL DEFAULT 1,
    -- 修正回数を追跡。初回提出=1、修正するたびにインクリメント

    revision_history JSONB DEFAULT '[]'::jsonb,
    -- 修正履歴
    -- [
    --   {
    --     "version": 1,
    --     "modified_at": "2026-03-28T17:30:00+09:00",
    --     "modified_by": "user_id_xxx",
    --     "changes": "引き継ぎ事項の納期を修正（金曜→木曜）"
    --   }
    -- ]

    submitted_at TIMESTAMPTZ,
    -- 最初に提出された日時

    -- === タイムスタンプ ===
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    -- === 一意性制約 ===
    CONSTRAINT unique_user_date UNIQUE (user_id, report_date)
    -- 1ユーザー1日1日報の制約
);

-- ============================================================
-- インデックス
-- ============================================================

-- 日付での検索用（社長が「3/28の日報」で検索する際に高速化）
CREATE INDEX idx_daily_reports_date
    ON daily_reports (report_date DESC);

-- ユーザー+日付の複合インデックス（特定スタッフの日報検索用）
CREATE INDEX idx_daily_reports_user_date
    ON daily_reports (user_id, report_date DESC);

-- ステータスでのフィルタリング用（未提出リマインド等）
CREATE INDEX idx_daily_reports_status
    ON daily_reports (status)
    WHERE status != 'archived';

-- 全文検索用インデックス（キーワード検索の高速化）
CREATE INDEX idx_daily_reports_summary_gin
    ON daily_reports USING gin (to_tsvector('japanese', summary));

-- 利用者の様子のJSONB検索用（特定利用者に関する記録の検索）
CREATE INDEX idx_daily_reports_client_notes_gin
    ON daily_reports USING gin (client_notes jsonb_path_ops);

-- 提出日時でのソート用
CREATE INDEX idx_daily_reports_submitted_at
    ON daily_reports (submitted_at DESC)
    WHERE submitted_at IS NOT NULL;

-- ============================================================
-- updated_at 自動更新トリガー
-- ============================================================

CREATE OR REPLACE FUNCTION update_daily_reports_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_daily_reports_updated_at
    BEFORE UPDATE ON daily_reports
    FOR EACH ROW
    EXECUTE FUNCTION update_daily_reports_updated_at();

-- ============================================================
-- RLS（Row Level Security）ポリシー
-- ============================================================

ALTER TABLE daily_reports ENABLE ROW LEVEL SECURITY;

-- スタッフ：自分の日報のみ閲覧・作成・更新可能
CREATE POLICY staff_own_reports ON daily_reports
    FOR ALL
    USING (
        user_id = auth.uid()
        AND EXISTS (
            SELECT 1 FROM users
            WHERE id = auth.uid()
            AND role IN ('staff', 'manager', 'owner')
        )
    );

-- 管理者：自分の日報 + 担当スタッフの日報を閲覧可能
CREATE POLICY manager_team_reports ON daily_reports
    FOR SELECT
    USING (
        EXISTS (
            SELECT 1 FROM users
            WHERE id = auth.uid()
            AND role = 'manager'
        )
        AND (
            user_id = auth.uid()
            OR user_id IN (
                SELECT staff_id FROM manager_staff_assignments
                WHERE manager_id = auth.uid()
            )
        )
    );

-- 社長：全日報を閲覧可能
CREATE POLICY owner_all_reports ON daily_reports
    FOR SELECT
    USING (
        EXISTS (
            SELECT 1 FROM users
            WHERE id = auth.uid()
            AND role = 'owner'
        )
    );
```

### 5.2 report_drafts テーブル（一時保存用）

```sql
-- ============================================================
-- 日報下書き（一時保存）テーブル
-- 作成途中の日報を一時的に保存する
-- ============================================================

CREATE TABLE report_drafts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,

    report_date DATE NOT NULL,

    -- 現在の質問番号（1〜5）
    current_question INTEGER NOT NULL DEFAULT 1
        CHECK (current_question BETWEEN 1 AND 5),

    -- 途中までの回答を保持
    answers JSONB NOT NULL DEFAULT '{}'::jsonb,
    -- 構造例:
    -- {
    --   "1": "午前中は利用者3名と...",
    --   "2": "Aさんは今日も集中して..."
    -- }

    -- 会話ログ
    conversation_log JSONB NOT NULL DEFAULT '[]'::jsonb,

    -- 有効期限（24時間後に自動削除）
    expires_at TIMESTAMPTZ NOT NULL DEFAULT (NOW() + INTERVAL '24 hours'),

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT unique_draft_user_date UNIQUE (user_id, report_date)
);

-- 期限切れの下書きを自動削除するためのインデックス
CREATE INDEX idx_report_drafts_expires
    ON report_drafts (expires_at);

-- ユーザーの下書き検索用
CREATE INDEX idx_report_drafts_user
    ON report_drafts (user_id, report_date DESC);
```

### 5.3 report_notifications テーブル（通知ログ用）

```sql
-- ============================================================
-- 日報通知ログテーブル
-- 送信した通知の記録を保持する（重複通知防止、監査用）
-- ============================================================

CREATE TABLE report_notifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 通知の種類
    notification_type VARCHAR(30) NOT NULL
        CHECK (notification_type IN (
            'submission',       -- 日報提出通知
            'reminder',         -- 未提出リマインド
            'daily_summary',    -- 日次サマリー
            'emergency_alert',  -- 緊急アラート
            'revision'          -- 修正通知
        )),

    -- 対象の日報（NULLの場合はサマリー通知等）
    report_id UUID REFERENCES daily_reports(id),

    -- 送信先ユーザー
    recipient_user_id UUID NOT NULL REFERENCES users(id),

    -- 通知内容
    message TEXT NOT NULL,

    -- 送信ステータス
    sent_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    delivered BOOLEAN DEFAULT FALSE,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_report_notifications_type_date
    ON report_notifications (notification_type, sent_at DESC);

CREATE INDEX idx_report_notifications_recipient
    ON report_notifications (recipient_user_id, sent_at DESC);
```

```typescript
// 日報関連メッセージのルーティング

async function handleDailyReportMessage(
  userId: string,
  message: string,
  currentState: ReportConversationState | null
): Promise<string> {

  // --- 1. 日報作成の開始判定 ---
  if (isDailyReportTrigger(message)) {
    return await startDailyReport(userId);
  }

  // --- 2. 日報の続き再開 ---
  if (isResumeTrigger(message) && currentState?.currentStep !== 'idle') {
    return await resumeDailyReport(userId, currentState);
  }

  // --- 3. キャンセル ---
  if (isCancelTrigger(message) && currentState?.currentStep !== 'idle') {
    return await cancelDailyReport(userId, currentState);
  }

  // --- 4. 進行中の質問への回答 ---
  if (currentState && isQuestionStep(currentState.currentStep)) {
    return await processAnswer(userId, message, currentState);
  }

  // --- 5. 確認中の応答 ---
  if (currentState?.currentStep === 'confirming') {
    if (isConfirmOk(message)) {
      return await saveDailyReport(userId, currentState);
    }
    if (isRevisionRequest(message)) {
      return await startRevision(userId, message, currentState);
    }
  }

  // --- 6. 修正中の応答 ---
  if (currentState?.currentStep === 'revising') {
    return await processRevision(userId, message, currentState);
  }

  // --- 7. 検索・閲覧（社長/管理者） ---
  if (isDailyReportSearchQuery(message)) {
    return await searchDailyReports(userId, message);
  }

  // --- 8. 該当なし ---
  return null; // 日報関連のメッセージではない
}

// トリガーワード判定
function isDailyReportTrigger(message: string): boolean {
  const triggers = [
    '日報', '日報書く', '日報を書く', 'にっぽう',
    '日報作成', '日報入力', '日報を入力',
  ];
  return triggers.some(t => message.includes(t));
}

function isCancelTrigger(message: string): boolean {
  const triggers = [
    'キャンセル', 'やめる', 'やめた', '中断',
    'あとで書く', 'あとで', '後で',
  ];
  return triggers.some(t => message.includes(t));
}

function isConfirmOk(message: string): boolean {
  const okPatterns = [
    'OK', 'ok', 'Ok', 'おk', 'おけ', 'オッケー',
    'はい', 'いいよ', 'いいです', '大丈夫',
    'それでいい', 'それで',
  ];
  return okPatterns.some(p =>
    message.trim().toLowerCase() === p.toLowerCase()
    || message.trim() === p
  );
```

```typescript
出力形式:

// 簡易パース（正規表現ベース）
function trySimpleParse(message: string): SearchParams | null {
  // 「○月○日の日報」パターン
  const dateMatch = message.match(
    /(\d{1,2})月(\d{1,2})日の日報/
  );
  if (dateMatch) {
    const month = dateMatch[1].padStart(2, '0');
    const day = dateMatch[2].padStart(2, '0');
    const year = new Date().getFullYear();
    const date = `${year}-${month}-${day}`;
    return { type: 'date', dateRange: { start: date, end: date } };
  }

  // 「○○さんの日報」パターン
  const personMatch = message.match(
    /(.+?)さんの(.*)日報/
  );
  if (personMatch) {
    return { type: 'person', personName: personMatch[1] };
  }

  // 「今月の日報まとめ」パターン
  if (message.includes('今月') && message.includes('まとめ')) {
    const now = new Date();
    const start = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}-01`;
    const end = now.toISOString().split('T')[0];
    return { type: 'period_summary', dateRange: { start, end } };
  }

  return null; // 簡易パースでは解釈不可
}
```

```typescript
      .lte('report_date', params.dateRange.end);

      config: 'japanese',

  query = query.limit(params.limit || 20);

  return data;
```

```typescript
  // ただし実際のスケジュールはvercel.jsonで設定

    await sendLineMessage(staff.line_user_id, message);

      notificationType: 'reminder',
      reportId: null,
      recipientUserId: staff.id,
      message,
    });
```

```
【処理】
```

```typescript
// 回答の最低品質チェック
function checkAnswerQuality(
  step: ReportStep,
  answer: string
): { acceptable: boolean; followUp?: string } {

  const trimmed = answer.trim();
  const emptyPatterns = [
    '特になし', 'とくになし', 'なし', 'ない', 'ありません',
    'なにもない', '何もない', 'nothing', '',
  ];

  const isEffectivelyEmpty = emptyPatterns.some(
    p => trimmed.toLowerCase() === p.toLowerCase()
  );

  // 質問1（作業内容）は空回答を許可しない
  if (step === 'q1_work_content' && isEffectivelyEmpty) {
    return {
      acceptable: false,
      followUp: '今日はお休みでしたか？\n業務日であれば、簡単でもよいので内容を教えていただけると助かります。（例：「事務作業」「ミーティング」等）',
    };
  }

  // 質問2（利用者の様子）で空の場合は確認
  if (step === 'q2_client_notes' && isEffectivelyEmpty) {
    return {
      acceptable: true,
      followUp: null, // 利用者と関わらない日もあるのでOK
    };
  }

  // その他の質問は空回答OK
  return { acceptable: true };
}
```

```typescript
// 日付判定ロジック
function determineReportDate(
  messageTimestamp: Date,
  userMessage: string
): { date: string; needsConfirmation: boolean; options?: string[] } {

  const jstHour = getJSTHour(messageTimestamp);
  const today = getJSTDateString(messageTimestamp);
  const yesterday = getJSTDateString(
    new Date(messageTimestamp.getTime() - 86400000)
  );

  // 「昨日の日報」等の明示的な日付指定
  if (userMessage.includes('昨日')) {
    return { date: yesterday, needsConfirmation: false };
  }

  // 深夜0:00〜5:00は確認が必要
  if (jstHour >= 0 && jstHour < 5) {
    return {
      date: null,
      needsConfirmation: true,
      options: [
        `1. 昨日（${formatDateJa(yesterday)}）の日報`,
        `2. 今日（${formatDateJa(today)}）の日報`,
      ],
    };
  }

  // 通常時間帯は当日
  return { date: today, needsConfirmation: false };
}
```
