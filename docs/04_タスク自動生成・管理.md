# 04. タスク自動生成・管理

## 1. 概要

### 1.1 この機能が解決する課題

就労継続支援事業所の現場では、日々の業務の中で「やるべきこと」が様々な形で発生する。日報に書かれた利用者の体調変化、外部からの電話対応、備品の不足、設備の故障、行政への書類提出 -- これらは口頭やメモで共有されることが多く、以下の深刻な問題を引き起こしている：

- **口頭伝達による対応漏れ**: 「○○さんに折り返し電話して」と言われたが、他の業務に追われて忘れてしまう
- **日報の情報が埋もれる**: 日報に「利用者Aさんの体調が悪い」と書いたが、翌日のフォローアクションにつながらない
- **期限管理の属人化**: 誰がいつまでに何をやるか、社長の頭の中にしかない
- **完了確認ができない**: 頼んだタスクが完了したのか、まだなのかが分からず再度確認する手間が発生
- **対応の抜け漏れが信頼を損なう**: 利用者や保護者、外部機関への対応が遅れることで事業所の信頼に影響

### 1.2 提供する価値

本機能は、AI秘書の中核機能として以下の価値を提供する：

1. **自動タスク抽出**: 日報やヒヤリハット報告の内容をClaude APIが解析し、アクションが必要な項目を自動的にタスク候補として提案する。スタッフが明示的にタスクを作成する手間がゼロになる
2. **自然言語によるタスク作成**: 社長が「田中に○○の書類を金曜までに作ってもらって」とLINEで送るだけで、担当者・内容・期限が設定されたタスクが即座に生成される
3. **期限管理・リマインド**: 期限当日の朝にリマインド通知、期限超過時には担当者と社長の双方に自動通知。対応漏れの発生をゼロにする
4. **進捗の可視化**: 社長は「今週のタスク」「期限超過のタスク」「田中のタスク」と聞くだけで、全体の進捗状況を即座に把握できる
5. **完了の自動検知**: 「○○株式会社に電話しました」という報告から、関連するタスクの完了を自動的に検知し、ステータスを更新する

## 2. ユーザーストーリー

### 2.1 スタッフ：日報からの自動タスク提案

```
スタッフとして、
日報をAIアシスト形式で書き終えたら、
AIが日報の内容を分析して「こんなタスクが必要ではないですか？」と提案してくれて、
承認するだけでタスクが作成・担当者に通知されるようにしたい。

受け入れ条件：
- 日報保存後、5秒以内にタスク候補が提案される
- 提案はカード形式で表示される（タイトル・担当者案・期限案）
- 「はい」で承認、「いいえ」でスキップ、「修正」で内容変更ができる
- 承認されたタスクは即座にDBに保存される
```

### 2.2 スタッフ：自分のタスク一覧確認

```
スタッフとして、
「自分のタスク」「今日やること」と送ったら、
自分に割り当てられた未完了タスクの一覧が見たい。

受け入れ条件：
- ステータス（未着手/進行中/完了）でグループ分けされている
- 期限超過のタスクは強調表示される
- タスク番号をタップするだけで完了報告できる
```

### 2.3 スタッフ：タスク完了報告

```
スタッフとして、
「○○のタスク完了しました」「○○に電話しました」と送ったら、
関連するタスクが自動的に完了ステータスに変わり、
タスク作成者に完了通知が届くようにしたい。

受け入れ条件：
- 自然言語から関連タスクが自動特定される
- 複数候補がある場合は選択肢が表示される
- 完了時にコメント（メモ）を追加できる
- 作成者にLINE通知が届く
```

### 2.4 管理者：担当利用者関連タスクの自動割り当て

```
管理者として、
自分の担当利用者に関するタスクが発生した場合、
自動的に自分に割り当てられるようにしたい。

受け入れ条件：
- 利用者テーブルに担当管理者が紐づいている
- 日報で利用者名に言及された場合、その担当管理者にタスクが割り当てられる
- 割り当て時にLINE通知が届く
- 管理者は自分の担当利用者のタスクのみ閲覧可能（社長は全件閲覧可能）
```

### 2.5 社長：全タスクの俯瞰

```
社長として、
「タスク一覧」「今週のタスク」「期限超過のタスク」「田中のタスク」と送ったら、
条件に合致するタスクの一覧を即座に確認したい。

受け入れ条件：
- 以下のフィルタに対応している
  - 期間：今日/今週/今月/全件
  - ステータス：未着手/進行中/完了/期限超過
  - 担当者：スタッフ名で指定
  - ソース：日報/ヒヤリハット/手動/カレンダー
- 一覧はコンパクトに整形されて表示される
- タスク数が多い場合は「上位10件」＋「残り○件あります」と要約される
```

### 2.6 社長：自然言語でのタスク手動作成

```
社長として、
「田中に○○の書類作成お願い。金曜まで」とLINEで送ったら、
AIが内容を解析して以下のタスクを自動生成し、確認を求めてほしい。
  - タスク名：○○の書類作成
  - 担当者：田中
  - 期限：今週金曜日
  - 優先度：中（デフォルト）

受け入れ条件：
- 担当者名はスタッフDBから自動マッチング
- 期限は相対日付（明日、来週月曜、月末等）に対応
- 確認メッセージで「OK」と答えると保存・通知される
- 「優先度高で」「急ぎで」と付けると優先度が自動調整される
```

### 2.7 社長：タスクの期限超過確認

```
社長として、
毎朝9時に自動で「期限超過タスク」の通知を受け取り、
対応が遅れている項目を即座に把握したい。

受け入れ条件：
- 期限超過タスクが1件以上ある場合のみ通知
- タスク名・担当者・超過日数が表示される
- 通知から直接担当者にリマインドを飛ばせる
```

## 4. 機能要件

### 4.1 必須機能（MVP）

| No. | 機能名 | 説明 | 入力 | 出力 | 優先度 |
|-----|--------|------|------|------|--------|
| F-01 | タスク自動抽出 | 日報テキストをClaude APIで解析し、アクション必要な項目をタスク候補として抽出する | 日報テキスト | タスク候補リスト（JSON） | 最高 |
| F-02 | タスク手動作成 | 社長の自然言語指示からタスクを作成する。担当者・期限・優先度を自動解析 | 自然言語テキスト | タスクレコード | 最高 |

```sql
            'daily_report',   -- 日報から自動抽出

            'manual',         -- 社長が手動作成
            'calendar',       -- カレンダー・定期タスクから
```

```typescript
  reportText: string,
  user: User
): Promise<TaskCandidate[]> {
  const prompt = TASK_EXTRACTION_PROMPT
    .replace('{{reporter_name}}', user.name)
    .replace('{{reporter_role}}', user.role)
    .replace('{{team_name}}', user.team_name || 'デフォルト')
    .replace('{{daily_report_text}}', reportText);

  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 2000,
    temperature: 0.1,  // 抽出精度を重視して低めに設定
    messages: [
      {
        role: 'user',
        content: prompt
      }
    ]
  });

  const content = response.content[0];
  if (content.type !== 'text') return [];

  try {
    const candidates: TaskCandidate[] = JSON.parse(content.text);
    // confidence が 0.5 未満のものは除外
    return candidates.filter(c => c.confidence >= 0.5);
  } catch (e) {
    console.error('タスク抽出のJSON解析に失敗:', e);
    return [];
  }
}
```

```typescript
async function buildTaskQuery(filters: TaskFilters): Promise<PostgrestFilterBuilder> {
  let query = supabase
    .from('tasks')
    .select(`
      id, title, description, status, priority,
      due_date, created_at, completed_at,
      assignee:users!assignee_id(id, name),
      creator:users!created_by(id, name),
      source
    `);

  // ステータスフィルタ
  if (filters.status && filters.status.length > 0) {
    query = query.in('status', filters.status);
  }

  // 担当者フィルタ
  if (filters.assignee_name) {
    // まずusersテーブルから担当者IDを取得
    const { data: users } = await supabase
      .from('users')
      .select('id')
      .ilike('name', `%${filters.assignee_name}%`);

    if (users && users.length > 0) {
      query = query.in('assignee_id', users.map(u => u.id));
    }
  }

  // 期間フィルタ
  if (filters.due_date_from) {
    query = query.gte('due_date', filters.due_date_from);
  }
  if (filters.due_date_to) {
    query = query.lte('due_date', filters.due_date_to);
  }

  // 期限超過フィルタ
  if (filters.is_overdue) {
    const today = new Date().toISOString().split('T')[0];
    query = query
      .lt('due_date', today)
      .in('status', ['pending', 'in_progress']);
  }

  // ソースフィルタ
  if (filters.source) {
    query = query.eq('source', filters.source);
  }

  // ソート
  const sortColumn = filters.sort_by || 'due_date';
  const sortAsc = (filters.sort_order || 'asc') === 'asc';
  query = query.order(sortColumn, { ascending: sortAsc });

  // リミット
  query = query.limit(filters.limit || 10);

  return query;
}
```

### 7.3 リマインドスケジュール（Cron処理）


```typescript
// /api/cron/task-reminders.ts
export async function GET(request: Request) {
  // 認証チェック（Vercel Cron Token）
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }

  const today = new Date().toISOString().split('T')[0];

  // 1. 期限当日のタスクを取得 → 担当者にリマインド
  const { data: todayTasks } = await supabase
    .from('tasks')
    .select('*, assignee:users!assignee_id(*)')
    .eq('due_date', today)
    .in('status', ['pending', 'in_progress']);

  // 担当者ごとにグループ化してLINE通知
  const groupedByAssignee = groupBy(todayTasks, 'assignee_id');
  for (const [assigneeId, tasks] of Object.entries(groupedByAssignee)) {
    await sendTodayReminder(tasks[0].assignee, tasks);
  }

  // 2. 期限超過タスクを取得 → 担当者＋社長に通知
  const { data: overdueTasks } = await supabase
    .from('tasks')
    .select('*, assignee:users!assignee_id(*)')
    .lt('due_date', today)
    .in('status', ['pending', 'in_progress'])
    .order('due_date', { ascending: true });

  if (overdueTasks && overdueTasks.length > 0) {
    // 担当者に個別通知
    const overdueByAssignee = groupBy(overdueTasks, 'assignee_id');
    for (const [assigneeId, tasks] of Object.entries(overdueByAssignee)) {
      await sendOverdueReminder(tasks[0].assignee, tasks);
    }

    // 社長にサマリー通知
    const owner = await getOwnerUser();
    await sendOverdueSummary(owner, overdueTasks);
  }

  // 3. リマインドログを記録
  for (const task of [...(todayTasks || []), ...(overdueTasks || [])]) {
    await supabase.from('task_comments').insert({
      task_id: task.id,
      user_id: 'SYSTEM',
      comment_type: 'reminder_sent',
      content: `自動リマインド送信（${today}）`
    });
    await supabase
      .from('tasks')
      .update({ remind_count: task.remind_count + 1 })
      .eq('id', task.id);
  }

  return Response.json({
    today_reminders: todayTasks?.length || 0,
    overdue_reminders: overdueTasks?.length || 0
  });
}
```

## 9. エッジケース・エラー処理

### 9.1 AIが過剰にタスクを抽出した場合

**状況**: 1つの日報から10件以上のタスク候補が抽出された場合

**対処方針**:
- confidence が 0.7 以上のもののみ提案する（閾値設定）
- 最大提案数を5件に制限する
- 5件を超える場合は「他にも○件の候補がありますが、確認しますか？」と段階的に提案

```typescript
const MAX_SUGGESTIONS_PER_REPORT = 5;
const MIN_CONFIDENCE_THRESHOLD = 0.7;

function filterAndLimitSuggestions(candidates: TaskCandidate[]): TaskCandidate[] {
  return candidates
    .filter(c => c.confidence >= MIN_CONFIDENCE_THRESHOLD)
    .sort((a, b) => b.confidence - a.confidence)
    .slice(0, MAX_SUGGESTIONS_PER_REPORT);
}
```

### 9.2 同じ内容のタスクが重複した場合

**状況**: 昨日の日報でも「A3用紙が少ない」、今日の日報でも「A3用紙が少ない」と書かれ、同じタスクが再提案される

**対処方針**:
- タスク作成前に、同一担当者の未完了タスクに対してタイトルの類似度チェックを実施
- 類似度が高い既存タスクがある場合は「既に似たタスクがあります」と通知

```typescript
async function checkDuplicateTask(
  title: string,
  assigneeId: string
): Promise<{ isDuplicate: boolean; existingTask?: Task }> {
  // 同一担当者の未完了タスクを取得
  const { data: openTasks } = await supabase
    .from('tasks')
    .select('*')
    .eq('assignee_id', assigneeId)
    .in('status', ['pending', 'in_progress']);

  if (!openTasks) return { isDuplicate: false };

  // Claude APIで類似度判定
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 500,
    messages: [{
      role: 'user',
      content: `新しいタスク「${title}」は、以下の既存タスクのいずれかと同じ内容ですか？
        ${openTasks.map(t => `- [${t.id}] ${t.title}`).join('\n')}
        同じ内容のタスクがあれば、そのIDを返してください。なければ "none" と返してください。`
    }]
  });

  const result = response.content[0].type === 'text' ? response.content[0].text.trim() : 'none';

  if (result === 'none') return { isDuplicate: false };

  const existingTask = openTasks.find(t => t.id === result);
  return { isDuplicate: !!existingTask, existingTask };
}
```

```typescript
async function safeExtractTasks(reportText: string, user: User): Promise<TaskCandidate[]> {
  const MAX_RETRIES = 2;

  for (let attempt = 0; attempt <= MAX_RETRIES; attempt++) {
    try {
      const response = await anthropic.messages.create({
        model: 'claude-sonnet-4-20250514',
        max_tokens: 2000,
        temperature: 0.1,
        messages: [{ role: 'user', content: buildPrompt(reportText, user) }]
      });

      const text = response.content[0].type === 'text' ? response.content[0].text : '';

      // JSON部分のみ抽出（前後のテキストがある場合に対応）
      const jsonMatch = text.match(/\[[\s\S]*\]/);
      if (!jsonMatch) {
        console.warn(`Attempt ${attempt}: JSON配列が見つかりません`);
        continue;
      }

      const candidates = JSON.parse(jsonMatch[0]);

      // バリデーション
      if (!Array.isArray(candidates)) {
        console.warn(`Attempt ${attempt}: 配列ではありません`);
        continue;
      }

      // 各候補の必須フィールド確認
      const validCandidates = candidates.filter(c =>
        c.title && typeof c.title === 'string' &&
        c.priority && ['high', 'medium', 'low'].includes(c.priority) &&
        typeof c.confidence === 'number'
      );

      return validCandidates;

    } catch (error) {
      console.error(`Attempt ${attempt}: タスク抽出エラー`, error);
      if (attempt === MAX_RETRIES) {
        // 最終リトライ失敗 → 空配列を返す（タスク提案なし）
        console.error('タスク抽出を断念しました');
        return [];
      }
    }
  }
  return [];
}
```

```sql
-- 統計クエリ例：今月の担当者別完了率
SELECT
    u.name AS assignee_name,
    COUNT(*) AS total_tasks,
    COUNT(*) FILTER (WHERE t.status = 'done') AS completed_tasks,
    ROUND(
        COUNT(*) FILTER (WHERE t.status = 'done') * 100.0 / NULLIF(COUNT(*), 0),
        1
    ) AS completion_rate,
    ROUND(
        AVG(
            EXTRACT(EPOCH FROM (t.completed_at - t.created_at)) / 86400
        ) FILTER (WHERE t.status = 'done'),
        1
    ) AS avg_days_to_complete
FROM tasks t
JOIN users u ON t.assignee_id = u.id
WHERE t.created_at >= DATE_TRUNC('month', CURRENT_DATE)
GROUP BY u.id, u.name
ORDER BY completion_rate DESC;
```

### 11.4 Notion連携（Phase 5以降）

タスク管理をNotionデータベースと双方向同期する拡張機能。社長がNotionのボード表示でタスクをドラッグ&ドロップで管理できるようになる。

**同期方針**:
- Supabaseのtasksテーブルがマスター（Source of Truth）
- Notionは閲覧・簡易操作用のビュー
- LINE経由の操作 → Supabase更新 → Notion同期（一方向）
- Notionでのステータス変更 → Webhook → Supabase更新（双方向）

```typescript
// Notion同期の概要
async function syncTaskToNotion(task: Task): Promise<void> {
  const notionPageId = task.notion_page_id;

  if (notionPageId) {
    // 既存ページを更新
    await notion.pages.update({
      page_id: notionPageId,
      properties: {
        'タスク名': { title: [{ text: { content: task.title } }] },
        'ステータス': { select: { name: statusToJapanese(task.status) } },
        '担当者': { people: [{ id: task.assignee_notion_id }] },
        '期限': { date: { start: task.due_date } },
        '優先度': { select: { name: priorityToJapanese(task.priority) } }
      }
    });
  } else {
    // 新規ページを作成
    const page = await notion.pages.create({
      parent: { database_id: TASK_DATABASE_ID },
      properties: { /* 上記と同じ */ }
    });
    // Notion Page IDをtasksテーブルに保存
    await supabase
      .from('tasks')
      .update({ notion_page_id: page.id })
      .eq('id', task.id);
  }
}
```

| 用語 | 説明 |
|------|------|
| タスク | アクションが必要な作業項目。担当者・期限・ステータスを持つ |

| エスカレーション | タスクが長期間未対応の場合に、上位者（社長）に報告を引き上げること |
| confidence | AIが抽出したタスクの確信度（0.00〜1.00）。0.7以上をデフォルトの採用閾値とする |
| 完了自動検知 | 会話の内容から既存タスクの完了を自動的に判定する機能 |
