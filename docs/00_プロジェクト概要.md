# AI秘書 プロジェクト概要

## 1. プロジェクトの目的

### 1.1 背景
さくら映像研は就労継続支援事業所（A型/B型）として、障がいを持つ利用者に対して作業機会の提供と就労支援を行っている。現在、以下の業務課題が存在する：

- **情報の分散**: 日報・出欠・支援記録がバラバラに管理されており、社長が全体を把握するのに時間がかかる
- **報告の負担**: スタッフが紙やExcelで日報を作成しており、記入に時間がかかる・書き忘れが発生する
- **対応の抜け漏れ**: 口頭で伝えられた対応事項がタスクとして管理されず、忘れられることがある
- **行政対応**: 支援計画の期限管理、実績報告書の提出など行政関連の期限管理が属人的
- **リアルタイム性の欠如**: 社長が現場の状況をリアルタイムで把握できない

### 1.2 解決するビジョン
LINE公式アカウントを通じて、AIがスタッフの日報作成を対話形式でアシストし、社長は「3月28日の日報」と送るだけで即座に情報を取得できる。さらに日報や会話の内容からAIが自動でタスクを抽出し、対応漏れを防止する。

### 1.3 プロジェクトのゴール
1. **スタッフの業務負担を軽減する**: 日報作成・記録業務をAIがアシストし、LINEで会話するだけで完了する
2. **社長の意思決定を高速化する**: 聞くだけで現場の状況が即座にわかる
3. **対応漏れをゼロにする**: AIがタスクを自動生成し、リマインドで追跡する
4. **行政対応を万全にする**: 提出期限・支援計画の見直し時期を自動管理する

## 2. システム全体構成

### 2.1 アーキテクチャ概要

```
┌─────────────┐         ┌──────────────┐         ┌──────────────┐
│  LINE公式    │ ──────→ │   Vercel     │ ──────→ │  Claude API  │
│  アカウント   │ ←────── │  (Webhook)   │ ←────── │  (AI処理)    │
└─────────────┘         └──────┬───────┘         └──────────────┘
                               │
                        ┌──────┴───────┐
                        │   Supabase   │
                        │  (データベース) │
                        └──────┬───────┘
                               │
                        ┌──────┴───────┐
                        │   Notion     │
                        │ (ナレッジ連携) │
                        └──────────────┘
```

### 2.2 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| フロントエンド | LINE Messaging API | ユーザーインターフェース |
| バックエンド | Vercel (Edge Functions) | Webhook受信・API処理 |
| AI | Claude API (Anthropic) | 自然言語処理・対話・タスク抽出 |
| データベース | Supabase (PostgreSQL) | 全データの永続化・リアルタイム |
| ナレッジ | Notion API | 議事録・ドキュメント連携 |
| 言語 | TypeScript | 全コードベース |

### 2.3 データフロー

```
スタッフがLINEで日報を送信
    ↓
Vercel Webhook が受信
    ↓
Claude API で内容を解析
    ↓
① 日報データをSupabaseに保存
② タスクを自動抽出してSupabaseに保存
③ 社長のLINEに要約を通知
④ 必要に応じて他スタッフに通知
```

## 3. ユーザーロール定義

### 3.1 社長（経営者）

| 項目 | 内容 |
|------|------|
| 人数 | 1名 |
| 主な利用シーン | 移動中・外出先から現場状況を確認 |
| 求める体験 | LINEで聞くだけで全てがわかる |
| 権限 | 全データの閲覧・全タスクの管理・スタッフ管理 |
| 通知を受けるもの | 日報要約、緊急アラート、期限超過タスク、行政期限 |

### 3.2 管理者（サービス管理責任者）

| 項目 | 内容 |
|------|------|
| 人数 | 2〜3名（想定） |
| 主な利用シーン | 現場で利用者対応をしながら記録 |
| 求める体験 | 会話するだけで日報が完成する |
| 権限 | 自分の担当利用者のデータ閲覧・日報作成・タスク管理 |
| 通知を受けるもの | 担当利用者の情報、自分のタスクリマインド |

### 3.3 スタッフ（職業指導員・生活支援員）

| 項目 | 内容 |
|------|------|
| 人数 | 3〜5名（想定） |
| 主な利用シーン | 作業終了後に日報を記録 |
| 求める体験 | 簡単に・短時間で日報が書ける |
| 権限 | 自分の日報作成・自分のタスク確認 |
| 通知を受けるもの | 日報リマインド、自分のタスクリマインド |

## 5. 用語集

| 用語 | 説明 |
|------|------|
| 利用者 | 就労支援事業所を利用する障がい者。スタッフではない |
| スタッフ | 事業所で働く職員（職業指導員・生活支援員） |
| 管理者 | サービス管理責任者。利用者の支援計画を作成する |
| 工賃 | 利用者に支払われる作業報酬 |
| 個別支援計画 | 利用者ごとに作成する支援目標・方法を定めた計画書 |
| モニタリング | 個別支援計画の進捗を定期的に確認・記録すること |
| アセスメント | 新規利用者の能力・希望・課題を評価すること |
| 実績報告書 | 毎月行政に提出する利用実績の報告書類 |
| ヒヤリハット | 事故には至らなかったが危険を感じた事例の報告 |
| A型事業所 | 雇用契約を結ぶ就労継続支援事業所（最低賃金保障） |
| B型事業所 | 雇用契約を結ばない就労継続支援事業所（工賃制） |
| 相談支援事業所 | 利用者を紹介してくれる外部の支援機関 |
| サービス等利用計画 | 相談支援事業所が作成する利用者の計画 |

## 7. 前提条件・制約

### 7.1 技術的前提
- LINE公式アカウント（Messaging API利用可能なプラン）が必要
- Vercelの無料プランで開始可能（月間利用量による）
- Claude APIのAPIキーが必要（従量課金）
- Supabaseの無料プランで開始可能（500MBまで）

### 7.2 運用上の制約
- LINE Messaging APIのメッセージ送信は月間の無料枠あり（200通/月が無料、超過は有料）
- AIの応答は完璧ではないため、重要なデータは人間が確認する前提
- 個人情報（利用者の氏名・障がい情報等）を扱うため、セキュリティに十分配慮する
- Supabaseの行レベルセキュリティ（RLS）を適切に設定する

### 7.3 セキュリティ要件
- 利用者の個人情報は暗号化して保存
- LINEのユーザーIDでアクセス制御
- ロール（社長/管理者/スタッフ）に応じたデータアクセス制限
- API通信はすべてHTTPS
- Supabase RLSで行レベルのアクセス制御
