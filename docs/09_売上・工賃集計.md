# 09: 売上・工賃集計

## 1. 概要

### 1.1 本機能の位置づけ

就労継続支援事業所（A型/B型）における経営の根幹は「売上の確保」と「利用者への適正な工賃支払い」である。本機能は、事業所の全売上を案件単位・取引先単位・月単位で自動集計し、利用者ごとの工賃を出席データと作業実績から正確に算出するシステムである。

社長が「今月の売上」とLINEで送るだけで、案件別の売上一覧・前月比・目標達成率を即座に返答し、経営判断に必要な情報をリアルタイムで提供する。

### 1.2 A型事業所とB型事業所の違い

| 項目 | A型（雇用型） | B型（非雇用型） |
|------|-------------|---------------|
| 雇用関係 | 利用者と雇用契約を結ぶ | 雇用契約なし |
| 賃金/工賃 | 最低賃金以上の給与を保障 | 工賃として支払い（最低賃金の適用なし） |
| 支払形態 | 月給 or 時給（最低賃金準拠） | 日額 or 出来高 or 月額 |
| 行政報告 | 平均賃金の報告義務あり | 平均工賃の報告義務あり |
| 目標 | 最低賃金の確実な支払い | 工賃向上（目標工賃の設定が必要） |

### 1.3 なぜ工賃管理が重要か

1. **行政への報告義務**: 毎年度、平均工賃（A型は平均賃金）を都道府県に報告する義務がある。虚偽報告は指定取消の対象となる
2. **事業所の評価に直結**: 報酬改定のたびに工賃実績が加算要件として重視される。工賃が高い事業所ほど報酬単価が上がる仕組み
3. **利用者の生活に直結**: B型事業所の全国平均工賃は月額約17,000円前後。わずかな改善でも利用者の生活に大きな影響を与える
4. **経営の持続可能性**: 売上と工賃のバランスが崩れると事業所の経営が成り立たない。売上以上の工賃は支払えないため、売上向上と工賃向上は表裏一体

## 2. ユーザーストーリー

### 2.1 社長（経営者）のストーリー

#### US-0901: 月次売上の即時確認
```
As a 社長
I want to LINEで「今月の売上」と送るだけで案件別の売上一覧を見たい
So that 事業所の経営状態をリアルタイムで把握できる

受け入れ条件:
- 「今月の売上」「売上どう？」「3月の売上」等の自然言語で発動する
- 案件別の売上一覧（案件名・金額・進捗率）が返る
- 合計金額が表示される
- 前月比（増減額・増減率）が表示される
- 月間目標に対する達成率が表示される
```

#### US-0902: 工賃状況の確認
```
As a 社長
I want to 「工賃どう？」と送るだけで利用者の平均工賃と目標との差を知りたい
So that 工賃向上の取り組みが順調か判断できる

受け入れ条件:
- 当月の平均工賃が表示される
- 目標工賃との差額が表示される
- 前月の平均工賃との比較が表示される
- 工賃が低い利用者のアラート（個人名は伏せてもよい）
- A型の場合は最低賃金を下回る利用者がいないか確認
```

#### US-0903: 月次比較・推移の確認
```
As a 社長
I want to 「先月と比べて」「前年同月比は？」と聞いて比較データを見たい
So that トレンドを把握して経営戦略を立てられる

受け入れ条件:
- 前月比の売上・工賃の増減を表示
- 前年同月比がある場合はそちらも表示
- テキストベースの簡易グラフ（棒グラフ的表現）
- 増減の主要因をAIが分析して提示
```

#### US-0904: 工賃改善の提案
```
As a 社長
I want to 「工賃上げるには？」と聞いてAIから改善提案をもらいたい
So that 具体的なアクションプランを立てられる

受け入れ条件:
- 現状の売上構成から単価交渉の余地を分析
- 作業効率化の可能性（出席率向上・作業速度改善等）を提示
- 新規案件獲得による影響をシミュレーション
- 過去の改善実績から効果の高い施策を提案
```

#### US-0905: 年次集計の確認
```
As a 社長
I want to 「今年度の実績まとめて」と言って行政報告用のデータを取得したい
So that 年度末の報告書作成がスムーズにできる

受け入れ条件:
- 年度（4月〜3月）の月別売上推移
- 年度の月別平均工賃推移
- 年間平均工賃（行政報告用の計算式に準拠）
- 利用者別の年間工賃一覧
```

### 2.2 管理者（サービス管理責任者等）のストーリー

#### US-0906: 担当利用者の工賃確認
```
As a 管理者
I want to 担当している利用者の工賃状況を一覧で確認したい
So that 個別支援計画の見直し材料にできる

受け入れ条件:
- 「担当の工賃確認」で担当利用者の一覧を表示
- 利用者ごとの今月の出席日数・作業時間・工賃見込みを表示
- 前月との比較が表示される
- 工賃が著しく低い利用者がいればアラート表示
```

#### US-0907: 利用者の工賃推移確認
```
As a 管理者
I want to 特定の利用者の工賃推移を確認したい
So that 支援計画のモニタリングに活用できる

受け入れ条件:
- 「〇〇さんの工賃推移」で過去6ヶ月分の推移を表示
- 月ごとの出席日数・作業時間・工賃を表示
- 増減傾向をAIが分析してコメント
```

### 2.3 スタッフのストーリー

```
スタッフは売上・工賃集計機能に直接アクセスしない。
スタッフが日々行う以下のアクションが、自動的に集計データに反映される：

- 日報での作業報告 → 作業時間・作業量として集計
- 出欠の記録（05_出欠管理） → 出席日数として集計
- 受注案件の進捗報告（06_受注・納品管理） → 売上データとして集計

つまり、スタッフは普段どおりLINEで報告するだけで、
売上・工賃のデータが自動的に蓄積される仕組みである。
```

| トリガー | 社長のLINEメッセージ / 月次バッチ処理 |
| 入力 | 年月（指定なしの場合は当月） |
| 処理 | orders + order_progress テーブルから該当月の売上を集計 |
| 出力 | 案件別売上一覧、合計、前月比、目標達成率 |
| 権限 | 社長のみ |

集計ロジック詳細:
```
売上 = Σ(各案件の金額 × 進捗率)
 ただし:
 - status = 'completed' の案件 → 金額の100%を売上計上
 - status = 'in_progress' の案件 → 金額 × 進捗率 を見込売上として計上
 - status = 'cancelled' の案件 → 集計から除外
 - 確定売上 = status が 'completed' かつ invoice_status が 'invoiced' or 'paid' のもの
 - 見込売上 = status が 'in_progress' のもの
```

#### F-0902: 工賃計算
| 項目 | 内容 |
|------|------|
| 機能ID | F-0902 |
| 機能名 | 利用者別工賃自動計算 |
| 説明 | 出席データと作業実績から利用者別の工賃を算出する |
| トリガー | 社長/管理者のLINEメッセージ / 月次バッチ処理 |
| 入力 | 年月、利用者ID（任意） |
| 処理 | 出席データ × 工賃単価で計算。A型は最低賃金チェック |
| 出力 | 利用者別工賃一覧、平均工賃、目標との差 |
| 権限 | 社長・管理者 |

工賃計算ロジック詳細:
```
【B型の工賃計算】
パターン1: 日額制
  月間工賃 = 出席日数 × 日額単価
  例: 20日 × ¥900 = ¥18,000

パターン2: 時間制
  月間工賃 = 作業時間 × 時間単価
  例: 80時間 × ¥230 = ¥18,400

パターン3: 出来高制
  月間工賃 = Σ(作業量 × 作業単価)
  例: テロップ入力500件 × ¥20 + 字幕作成200件 × ¥30 = ¥16,000

パターン4: 混合制（基本日額 + 出来高）
  月間工賃 = (出席日数 × 基本日額) + Σ(作業量 × 作業単価)
  例: 20日 × ¥500 + テロップ入力500件 × ¥16 = ¥18,000

【A型の賃金計算】
  月間賃金 = 勤務時間 × 時給
  ※ 時給 >= 都道府県の最低賃金 であることを必ずチェック
  例: 80時間 × ¥1,054 = ¥84,320
```

### 5.3 集計用ビュー

```sql
-- 月次売上サマリービュー
CREATE VIEW v_monthly_revenue_summary AS
SELECT
    year_month,
    SUM(amount) AS total_revenue,
    SUM(CASE WHEN revenue_type = 'confirmed' THEN amount ELSE 0 END) AS confirmed_revenue,
    SUM(CASE WHEN revenue_type = 'expected' THEN amount ELSE 0 END) AS expected_revenue,
    SUM(CASE WHEN revenue_type = 'paid' THEN amount ELSE 0 END) AS paid_revenue,
    COUNT(DISTINCT order_id) AS order_count,
    SUM(CASE WHEN category = 'order' THEN amount ELSE 0 END) AS order_revenue,
    SUM(CASE WHEN category = 'subsidy' THEN amount ELSE 0 END) AS subsidy_revenue,
    SUM(CASE WHEN category = 'other' THEN amount ELSE 0 END) AS other_revenue
FROM monthly_revenue
GROUP BY year_month
ORDER BY year_month DESC;

-- 月次工賃サマリービュー
CREATE VIEW v_monthly_wage_summary AS
SELECT
    wr.year_month,
    wr.employment_type,
    COUNT(*) AS client_count,
    AVG(wr.total_wage)::INTEGER AS average_wage,
    MIN(wr.total_wage) AS min_wage,
    MAX(wr.total_wage) AS max_wage,
    SUM(wr.total_wage) AS total_wage_paid,
    AVG(wr.work_days)::DECIMAL(4,1) AS average_work_days,
    AVG(wr.work_hours)::DECIMAL(6,1) AS average_work_hours
FROM wage_records wr
WHERE wr.status IN ('calculated', 'approved', 'paid')
GROUP BY wr.year_month, wr.employment_type
ORDER BY wr.year_month DESC, wr.employment_type;

-- 利用者別工賃推移ビュー
CREATE VIEW v_client_wage_trend AS
SELECT
    wr.client_id,
    c.name AS client_name,
    wr.year_month,
    wr.employment_type,
    wr.work_days,
    wr.work_hours,
    wr.total_wage,
    wr.hourly_equivalent,
    wr.status,
    LAG(wr.total_wage) OVER (
        PARTITION BY wr.client_id ORDER BY wr.year_month
    ) AS prev_month_wage,
    wr.total_wage - COALESCE(
        LAG(wr.total_wage) OVER (
            PARTITION BY wr.client_id ORDER BY wr.year_month
        ), 0
    ) AS wage_change
FROM wage_records wr
JOIN clients c ON c.id = wr.client_id
ORDER BY wr.client_id, wr.year_month;

-- 売上と工賃のバランスチェックビュー
CREATE VIEW v_revenue_wage_balance AS
SELECT
    r.year_month,
    r.total_revenue,
    w_b.total_wage_paid AS type_b_total_wage,
    w_a.total_wage_paid AS type_a_total_wage,
    COALESCE(w_b.total_wage_paid, 0) + COALESCE(w_a.total_wage_paid, 0) AS total_wage,
    r.total_revenue - COALESCE(w_b.total_wage_paid, 0) - COALESCE(w_a.total_wage_paid, 0) AS balance,
    CASE
        WHEN r.total_revenue > 0 THEN
            ROUND(
                (COALESCE(w_b.total_wage_paid, 0) + COALESCE(w_a.total_wage_paid, 0))::DECIMAL
                / r.total_revenue * 100, 1
            )
        ELSE 0
    END AS wage_ratio_percent
FROM v_monthly_revenue_summary r
LEFT JOIN v_monthly_wage_summary w_b
    ON r.year_month = w_b.year_month AND w_b.employment_type = 'type_b'
LEFT JOIN v_monthly_wage_summary w_a
    ON r.year_month = w_a.year_month AND w_a.employment_type = 'type_a'
ORDER BY r.year_month DESC;
```

```sql
        o.total_amount,
        o.status,
        COALESCE(op.progress_rate, 0) AS progress_rate,
        CASE
            WHEN o.status = 'completed' THEN o.total_amount
            WHEN o.status = 'in_progress' THEN
                ROUND(o.total_amount * COALESCE(op.progress_rate, 0) / 100.0)
            ELSE 0
        END AS calculated_revenue,
        CASE
            WHEN o.status = 'completed' AND o.invoice_status IN ('invoiced', 'paid')
                THEN 'confirmed'
            WHEN o.status = 'completed' AND o.invoice_status = 'not_invoiced'
                THEN 'invoiced'
            WHEN o.status = 'in_progress'
                THEN 'expected'
            ELSE 'confirmed'
        END AS revenue_type
    FROM orders o
    LEFT JOIN (
        SELECT order_id, MAX(progress_rate) AS progress_rate
        FROM order_progress
        GROUP BY order_id
    ) op ON o.id = op.order_id
    WHERE
        o.status IN ('completed', 'in_progress')
        AND (
            -- 納品月が指定月 OR 制作中で予定月が指定月
            TO_CHAR(o.delivered_at, 'YYYY-MM') = $1  -- $1 = '2026-03'
            OR (o.status = 'in_progress'
                AND TO_CHAR(o.deadline, 'YYYY-MM') = $1)
        )
)
SELECT
    order_id,
    client_company_name,
    order_title,
    total_amount,
    status,
    progress_rate,
    calculated_revenue,
    revenue_type
FROM order_revenue
ORDER BY calculated_revenue DESC;

-- 月次売上サマリー（前月比つき）
SELECT
    cur.year_month,
    cur.total_revenue AS current_revenue,
    prev.total_revenue AS prev_revenue,
    cur.total_revenue - COALESCE(prev.total_revenue, 0) AS revenue_change,
    CASE
        WHEN COALESCE(prev.total_revenue, 0) > 0
        THEN ROUND(
            (cur.total_revenue - prev.total_revenue)::DECIMAL
            / prev.total_revenue * 100, 1
        )
        ELSE NULL
    END AS change_rate_percent,
    rt.target_amount,
    CASE
        WHEN COALESCE(rt.target_amount, 0) > 0
        THEN ROUND(
            cur.total_revenue::DECIMAL / rt.target_amount * 100, 1
        )
        ELSE NULL
    END AS achievement_rate_percent
FROM v_monthly_revenue_summary cur
LEFT JOIN v_monthly_revenue_summary prev
    ON prev.year_month = TO_CHAR(
        (cur.year_month || '-01')::DATE - INTERVAL '1 month', 'YYYY-MM'
    )
LEFT JOIN revenue_targets rt
    ON rt.fiscal_year = EXTRACT(YEAR FROM (cur.year_month || '-01')::DATE)
    AND rt.month = EXTRACT(MONTH FROM (cur.year_month || '-01')::DATE)
WHERE cur.year_month = $1;
```

```sql
-- B型利用者の工賃計算（日額制の場合）
WITH attendance_summary AS (
    SELECT
        client_id,
        COUNT(*) FILTER (WHERE status = 'present') AS work_days,
        SUM(
            EXTRACT(EPOCH FROM (check_out_time - check_in_time)) / 3600.0
        ) FILTER (WHERE status = 'present') AS work_hours
    FROM attendance
    WHERE TO_CHAR(date, 'YYYY-MM') = $1  -- $1 = '2026-03'
    GROUP BY client_id
),
piece_rate_summary AS (
    SELECT
        client_id,
        SUM(amount) AS piece_rate_total
    FROM piece_rate_records
    WHERE TO_CHAR(work_date, 'YYYY-MM') = $1
    GROUP BY client_id
)
SELECT
    c.id AS client_id,
    c.name AS client_name,
    wc.employment_type,
    wc.wage_type,
    COALESCE(att.work_days, 0) AS work_days,
    COALESCE(att.work_hours, 0)::DECIMAL(6,2) AS work_hours,
    -- 基本工賃の計算
    CASE wc.wage_type
        WHEN 'daily' THEN COALESCE(att.work_days, 0) * wc.daily_rate
        WHEN 'hourly' THEN ROUND(COALESCE(att.work_hours, 0) * wc.hourly_rate)
        WHEN 'piece_rate' THEN 0  -- 出来高のみ
        WHEN 'mixed' THEN COALESCE(att.work_days, 0) * wc.base_daily_rate
    END AS base_wage,
    -- 出来高工賃
    COALESCE(pr.piece_rate_total, 0) AS piece_rate_wage,
    -- 合計工賃
    CASE wc.wage_type
        WHEN 'daily' THEN COALESCE(att.work_days, 0) * wc.daily_rate
        WHEN 'hourly' THEN ROUND(COALESCE(att.work_hours, 0) * wc.hourly_rate)
        WHEN 'piece_rate' THEN COALESCE(pr.piece_rate_total, 0)
        WHEN 'mixed' THEN
            COALESCE(att.work_days, 0) * wc.base_daily_rate
            + COALESCE(pr.piece_rate_total, 0)
    END AS total_wage,
    -- 時給換算（A型の最低賃金チェック用）
    CASE
        WHEN COALESCE(att.work_hours, 0) > 0 THEN
            ROUND(
                CASE wc.wage_type
                    WHEN 'daily' THEN COALESCE(att.work_days, 0) * wc.daily_rate
                    WHEN 'hourly' THEN ROUND(COALESCE(att.work_hours, 0) * wc.hourly_rate)
                    WHEN 'piece_rate' THEN COALESCE(pr.piece_rate_total, 0)
                    WHEN 'mixed' THEN
                        COALESCE(att.work_days, 0) * wc.base_daily_rate
                        + COALESCE(pr.piece_rate_total, 0)
                END::DECIMAL / att.work_hours
            )
        ELSE 0
    END AS hourly_equivalent,
    -- 最低賃金割れチェック（A型のみ）
    CASE
        WHEN wc.employment_type = 'type_a'
             AND COALESCE(att.work_hours, 0) > 0
             AND ROUND(
                 (COALESCE(att.work_hours, 0) * wc.hourly_rate)::DECIMAL
                 / att.work_hours
             ) < wc.minimum_wage
        THEN TRUE
        ELSE FALSE
    END AS minimum_wage_violation
FROM clients c
JOIN active_wage_config wc ON wc.client_id = c.id
LEFT JOIN attendance_summary att ON att.client_id = c.id
LEFT JOIN piece_rate_summary pr ON pr.client_id = c.id
WHERE c.status = 'active'
ORDER BY wc.employment_type, total_wage DESC;
```

#### 6.2.3 行政報告用の年間平均工賃計算

```sql
-- 行政報告用: 年度平均工賃の計算
-- 計算式: 年間工賃総額 ÷ 年間延べ利用者数
-- ※ 延べ利用者数 = 各月の利用者数の合計
WITH fiscal_year_months AS (
    -- 年度の月一覧（4月〜翌3月）
    SELECT generate_series(
        ($1 || '-04-01')::DATE,  -- $1 = 年度（例: 2025）
        (($1::INTEGER + 1) || '-03-01')::DATE,
        INTERVAL '1 month'
    )::DATE AS month_start
),
monthly_data AS (
    SELECT
        TO_CHAR(fym.month_start, 'YYYY-MM') AS year_month,
        wr.employment_type,
        COUNT(*) AS client_count,
        SUM(wr.total_wage) AS total_wage
    FROM fiscal_year_months fym
    JOIN wage_records wr ON wr.year_month = TO_CHAR(fym.month_start, 'YYYY-MM')
    WHERE wr.status IN ('calculated', 'approved', 'paid')
    GROUP BY TO_CHAR(fym.month_start, 'YYYY-MM'), wr.employment_type
)
SELECT
    employment_type,
    SUM(total_wage) AS annual_total_wage,
    SUM(client_count) AS annual_total_clients,  -- 延べ利用者数
    ROUND(SUM(total_wage)::DECIMAL / NULLIF(SUM(client_count), 0))
        AS annual_average_wage
FROM monthly_data
GROUP BY employment_type;
```

### 8.1 月途中の入退所者の工賃計算

```
【ケース】利用者が月の途中で入所、または退所した場合

入所日: 3月15日の場合
- 出席可能日数: 15日〜31日の営業日のみ（例: 12日）
- 工賃計算: 実際の出席日数で計算（日割り計算）
- 平均工賃計算への影響:
  方式A: 月途中入所者も含めて平均を計算（在籍日数按分なし）
  方式B: 在籍日数で按分して平均に含める（行政報告では通常こちら）

退所日: 3月10日の場合
- 出席可能日数: 1日〜10日の営業日のみ（例: 7日）
- 工賃計算: 実際の出席日数で計算
- 退所月の工賃は通常通り計算し、支払済にする

【実装】
```

```sql
-- 月途中入退所者の在籍日数を考慮した平均工賃計算
WITH client_tenure AS (
    SELECT
        c.id AS client_id,
        c.enrollment_date,
        c.discharge_date,
        -- 当月の在籍開始日
        GREATEST(
            c.enrollment_date,
            ($1 || '-01')::DATE  -- 月初
        ) AS tenure_start,
        -- 当月の在籍終了日
        LEAST(
            COALESCE(c.discharge_date, ($1 || '-01')::DATE + INTERVAL '1 month' - INTERVAL '1 day'),
            ($1 || '-01')::DATE + INTERVAL '1 month' - INTERVAL '1 day'  -- 月末
        ) AS tenure_end
    FROM clients c
    WHERE c.status IN ('active', 'discharged')
      AND c.enrollment_date <= ($1 || '-01')::DATE + INTERVAL '1 month' - INTERVAL '1 day'
      AND (c.discharge_date IS NULL OR c.discharge_date >= ($1 || '-01')::DATE)
),
tenure_ratio AS (
    SELECT
        client_id,
        -- 営業日ベースの在籍割合
        (tenure_end - tenure_start + 1)::DECIMAL /
        (($1 || '-01')::DATE + INTERVAL '1 month' - ($1 || '-01')::DATE)::INTEGER AS ratio
    FROM client_tenure
)
SELECT
    wr.employment_type,
    -- 在籍日数按分した平均工賃
    ROUND(
        SUM(wr.total_wage / GREATEST(tr.ratio, 0.1))  -- ratioで按分（フル月に換算）
        / COUNT(*)
    ) AS adjusted_average_wage,
    -- 単純平均（参考値）
    AVG(wr.total_wage)::INTEGER AS simple_average_wage
FROM wage_records wr
JOIN tenure_ratio tr ON tr.client_id = wr.client_id
WHERE wr.year_month = $1
GROUP BY wr.employment_type;
```

### 8.2 A型の最低賃金割れ検知

```typescript
interface MinimumWageViolation {
  clientId: string;
  clientName: string;
  workHours: number;
  totalWage: number;
  hourlyEquivalent: number;
  minimumWage: number;
  shortfall: number;       // 不足額
  correctedWage: number;   // 最低賃金で再計算した場合の金額
}

async function checkMinimumWageViolations(
  yearMonth: string
): Promise<MinimumWageViolation[]> {
  const result = await supabase.rpc('check_minimum_wage', {
    target_month: yearMonth
  });

  // 違反がある場合は即座に緊急通知
  if (result.data && result.data.length > 0) {
    for (const violation of result.data) {
      // 工賃を最低賃金ベースに自動修正（ドラフト状態で）
      await supabase
        .from('wage_records')
        .update({
          total_wage: violation.correctedWage,
          hourly_equivalent: violation.minimumWage,
          note: `最低賃金割れを検知し自動修正。修正前: ¥${violation.totalWage}`,
          status: 'draft',  // 承認待ちに戻す
        })
        .match({ client_id: violation.clientId, year_month: yearMonth });
    }
  }

  return result.data || [];
}
```

```sql
-- 最低賃金チェック用のストアドファンクション
CREATE OR REPLACE FUNCTION check_minimum_wage(target_month VARCHAR)
RETURNS TABLE (
    client_id UUID,
    client_name VARCHAR,
    work_hours DECIMAL,
    total_wage INTEGER,
    hourly_equivalent INTEGER,
    minimum_wage INTEGER,
    shortfall INTEGER,
    corrected_wage INTEGER
) AS $$
BEGIN
    RETURN QUERY
    SELECT
        wr.client_id,
        c.name AS client_name,
        wr.work_hours,
        wr.total_wage,
        wr.hourly_equivalent,
        wc.minimum_wage,
        (wc.minimum_wage * wr.work_hours)::INTEGER - wr.total_wage AS shortfall,
        (wc.minimum_wage * wr.work_hours)::INTEGER AS corrected_wage
    FROM wage_records wr
    JOIN clients c ON c.id = wr.client_id
    JOIN active_wage_config wc ON wc.client_id = wr.client_id
    WHERE wr.year_month = target_month
      AND wr.employment_type = 'type_a'
      AND wr.work_hours > 0
      AND wr.hourly_equivalent < wc.minimum_wage;
END;
$$ LANGUAGE plpgsql;
```

```typescript
interface UninvoicedOrder {

    ),
  }));
}
```

### 8.4 売上と工賃のバランスチェック

```typescript
interface RevenueWageBalance {
  yearMonth: string;
  totalRevenue: number;
  totalWage: number;
  wageRatioPercent: number;
  isWarning: boolean;      // 80%超
  isCritical: boolean;     // 100%超（赤字確定）
  breakEvenRevenue: number; // 工賃をまかなうために必要な最低売上
}

async function checkRevenueWageBalance(
  yearMonth: string
): Promise<RevenueWageBalance> {
  const { data } = await supabase
    .from('v_revenue_wage_balance')
    .select('*')
    .eq('year_month', yearMonth)
    .single();

  const totalRevenue = data?.total_revenue || 0;
  const totalWage = data?.total_wage || 0;
  const ratio = totalRevenue > 0
    ? (totalWage / totalRevenue) * 100
    : 0;

  return {
    yearMonth,
    totalRevenue,
    totalWage,
    wageRatioPercent: Math.round(ratio * 10) / 10,
    isWarning: ratio > 80,
    isCritical: ratio > 100,
    breakEvenRevenue: Math.ceil(totalWage / 0.8),  // 工賃比率80%を基準
  };
}
```

### 8.5 その他のエッジケース

| ケース | 対応方針 |
|--------|----------|
| 出席データが未入力の利用者がいる | 工賃計算時に警告を出し、0日として仮計算。管理者に確認を促す |
| 案件の金額が0円または未設定 | 売上集計から除外し、データ不足として社長に通知 |
| 同一案件が複数月にまたがる | 進捗率の差分で月ごとの売上を按分。例: 先月50%→今月80%なら今月分は30%分 |
| 利用者の工賃設定が未登録 | 工賃計算をスキップし、管理者に設定を促す通知を送信 |
| 最低賃金が都道府県で改定された | wage_config のminimum_wageを更新する運用。改定日以降の計算に反映 |
| 賞与・特別手当の支給 | wage_records の bonus フィールドに手動入力。月次工賃とは別扱い |
| 閏年・月末日の違い | 営業日カレンダーに基づいて計算。2月の工賃が少ないのは正常 |
| データの遡及修正 | 過去月のデータを修正した場合、年間平均工賃を再計算して更新 |
